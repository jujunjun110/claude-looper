# stub-pages: 全ルートのスタブページ作成

## タスク ID: stub-pages

全 7 ルートのスタブページを Server Component として作成し、app-shell レイアウトでラップ。

## 作成・変更ファイル
- `src/components/app-sidebar.tsx` — サイドバーナビゲーション（Client Component）
- `src/app/(app)/layout.tsx` — app-shell レイアウト（SidebarProvider + Sidebar + SidebarInset）
- `src/app/(app)/page.tsx` — ダッシュボード
- `src/app/(app)/checks/new/page.tsx` — 新規チェック
- `src/app/(app)/checks/[id]/page.tsx` — チェック結果詳細
- `src/app/(app)/rules/page.tsx` — 表現ルール管理
- `src/app/(app)/knowledge/page.tsx` — ナレッジ記事管理
- `src/app/(app)/knowledge/import/page.tsx` — ナレッジ取込
- `src/app/login/page.tsx` — ログイン（app-shell 外）
- `src/app/page.tsx` — 削除（(app) ルートグループに移動）

## 設計判断
- `(app)` ルートグループで認証済みページを囲み、app-shell レイアウトを適用
- `/login` は `(auth)` ではなく `app/login/` 直下に配置（シンプルさ優先、今後 middleware で認証制御）
- shadcn/ui の Sidebar コンポーネントを活用し、SidebarProvider → Sidebar + SidebarInset 構成
- チェック詳細ページは `params: Promise<{ id: string }>` で Next.js 15 の非同期 params に対応

## 次のタスクへの申し送り
- 各スタブページは h1 タイトルのみ。実際の UI 実装は後続タスクで行う
- middleware.ts による認証チェック・リダイレクトは auth 関連タスクで実装予定
