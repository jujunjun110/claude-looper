# prisma-schema: Prisma スキーマ全7テーブル定義

- **タスク ID**: prisma-schema
- **内容**: `apps/content-reviewer/prisma/schema.prisma` に全7テーブル・5enum・pgvector拡張・リレーション・インデックスを定義

## 作成・変更ファイル
- `apps/content-reviewer/prisma/schema.prisma` — 新規作成（スキーマ本体）
- `apps/content-reviewer/package.json` — prisma/client 追加、prisma:generate スクリプト追加
- `pnpm-lock.yaml` — 依存追加に伴う更新

## 設計判断
- pgvector の `vector(1536)` は Prisma が未ネイティブサポートのため `Unsupported("vector(1536)")` で定義。`previewFeatures = ["postgresqlExtensions"]` で拡張を有効化
- カラム名は snake_case（`@map`）、モデルフィールドは camelCase（Prisma 規約準拠）
- Prisma 5.22.0 を採用（Node.js v22.1.0 との互換性のため、最新版は 22.12+ を要求）
- prisma は devDependencies、@prisma/client は dependencies に配置

## 次のタスクへの申し送り
- `KnowledgeEmbedding.embedding` は `Unsupported` 型のため、Repository 実装時に `$queryRaw` でベクトル検索を行う必要あり
- DB マイグレーション（`prisma migrate dev`）は DATABASE_URL 設定後に別タスクで実行
