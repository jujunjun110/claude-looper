# login-page-ui: ログインページUI・ログアウトボタン実装

## タスク
Google OAuth ログインボタン、ログアウトボタン、ユーザー情報表示の実装

## 作成・変更ファイル
- `src/backend/contexts/auth/presentation/actions/sign-in.action.ts` — Server Action: Supabase Auth の Google OAuth フロー開始
- `src/backend/contexts/auth/presentation/actions/sign-out.action.ts` — Server Action: サインアウト + /login リダイレクト
- `src/backend/contexts/auth/presentation/loaders/current-user.loader.ts` — Loader: Supabase session から User ドメインモデルへ変換
- `src/app/auth/callback/route.ts` — OAuth コールバック API Route: code → session 交換
- `src/components/auth/login-button.tsx` — Client Component: form action で sign-in.action 呼び出し
- `src/components/auth/logout-button.tsx` — Client Component: form action で sign-out.action 呼び出し
- `src/app/login/page.tsx` — LoginButton 配置、アプリ名・説明テキスト追加
- `src/app/(app)/layout.tsx` — ヘッダー右側にユーザー名表示 + LogoutButton 追加

## 設計判断
- auth presentation 層は Supabase Auth の薄いラッパー。UseCase/composition 層は不要（OAuth はリダイレクトベースで複雑なビジネスロジックがない）
- sign-in は Server Action 内で `redirect()` を使い OAuth URL へ遷移。クライアント側は form action で呼び出す
- OAuth callback は API Route（外部リダイレクト受信のため、Server Action では不可）
- current-user.loader は `User.fromSupabaseUser()` でドメインモデルに変換し、型安全を維持
