# app-layout-shell: アプリレイアウトシェル実装

- **タスクID**: app-layout-shell
- **内容**: ルートレイアウトに shadcn/ui Sidebar ベースのシェル UI を実装

## 作成・変更ファイル
- `src/components/layout/header.tsx` — ヘッダー（SidebarTrigger + アプリ名）
- `src/components/layout/sidebar-nav.tsx` — サイドバーナビ（4項目: ダッシュボード, チェック実行, 表現ルール, ナレッジ）
- `src/components/layout/app-shell.tsx` — SidebarProvider + SidebarNav + Header + main を組み合わせるシェル
- `src/app/layout.tsx` — AppShell を組み込み
- `src/app/page.tsx` — 不要な `<main>` ラッパーを除去

## 設計判断
- shadcn/ui の SidebarProvider → Sidebar + SidebarInset パターンを採用。レスポンシブ対応は Sidebar コンポーネント内蔵の Sheet（モバイル時）で実現
- `usePathname` でアクティブ状態を判定。ダッシュボード(`/`)は完全一致、他はプレフィックス一致
- 全レイアウトコンポーネントは `'use client'` （SidebarProvider が React Context を使用するため）

## 次のタスクへの申し送り
- 認証未実装のため、全ページにレイアウトが適用される。`/login` ページ実装時はレイアウトの出し分けが必要
