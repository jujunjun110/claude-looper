## タスク: auth-middleware-and-callback

認証ミドルウェアと OAuth コールバックルートの実装。

### 作成ファイル
- `apps/content-reviewer/src/middleware.ts` — Next.js middleware（認証チェック）
- `apps/content-reviewer/src/app/api/auth/callback/route.ts` — Supabase Auth code exchange

### 設計判断
- middleware は既存の `createSupabaseMiddlewareClient` を利用し、`supabase.auth.getUser()` でセッション検証
- パブリックパス（/login, /api/auth/callback）とスタティックリソースは matcher + isPublicPath の二重チェックでスキップ
- `NODE_ENV=development` 時は認証チェックを完全スキップ（Supabase 接続不要で開発可能に）
- callback route は `createSupabaseServerClient` で code exchange し、成功時はルートへ、失敗時は /login へリダイレクト

### 次のタスクへの申し送り
- ログインページに Google OAuth ボタンの実装が必要（`supabase.auth.signInWithOAuth` を呼ぶ UI）
- ユーザー同期処理（Supabase auth.users → アプリの users テーブル）は別タスクで実装
